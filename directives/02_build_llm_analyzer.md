# Directive 02: Build the AI Analyst (LLM Integration)

**Goal:** Write a Python script that reads the parsed network data and uses the Gemini API to generate human-readable insights and a Mermaid.js topology diagram.

**Execution Steps:**
1. Write a script named `llm_analyzer.py` in the `/execution/` directory.
2. The script must read the `/.tmp/flows_summary.json` file generated by the previous step.
3. Use the `google-generativeai` Python library. Configure it to securely load the `GEMINI_API_KEY` from the `.env` file in the root directory (using a library like `python-dotenv`).
4. Set the generative model specifically to `gemini-2.5-flash`.
5. Construct a prompt asking the LLM to act as a network security expert, analyze the JSON flows, provide 3-4 bullet points of insights (like drops, heavy traffic, or anomalies), and output the code for a Mermaid.js graph showing the communication.
6. Save the LLM's structured output into a file named `analysis_result.json` inside the `/.tmp/` directory. It should clearly separate the text `insights` and the `mermaid_code`.
7. Run the script. **Self-Anneal Rule:** Fix any API connection errors, JSON parsing errors, or missing dependencies (like `google-generativeai` or `python-dotenv`) until the output file is generated successfully.
